# 高级和正则

## 迭代器

是一个实现了\_\_iter\_\_和\_\_next\_\_方法的对象,使得可以逐步遍历它的对元素
特点：

1. 手动管理：需要显示的实现\_\_iter\_\_和\_\_next\_\_方法
2. 状态管理：迭代器需要自己管理迭代的状态，包括当前位置和结束位置。
3. 内存使用：内存使用取决于迭代器的实现，通常是惰性计算（即按需生成数据）

```python
# 迭代器低层实现
class MyIterator:
    def __init__(self,start,end):
        self.start = start
        self.end = end
    def __iter__(self):
        return self
    def __next__(self):
        if self.start < self.end:
            value = self.start
            self.start += 1
            return value
        else:
            raise StopIteration # raise 用于抛出异常，StopIteration 用于表示迭代器已经结束

my_iterator = MyIterator(0,10)
for i in my_iterator:
    print(i)
```

## 生成器

根据制定的规则循环生成数据，当条件不成立时则生成数据结束。数据不是一次性生成的，而是使用一个再生成一个。
作用：可以节省大量的内存。  
创建生成器方式：

1. 生成器推导式
2. yield 关键字

yield 关键字：

1. 创建生成器对象
2. 把值存储到生成器中
3. 返回生成器

## 迭代器和生成器的区别

1. 实现方式
   - 迭代器：需要**inter**()和**next**()方法，手动管理迭代状态
   - 生成器：通过 yield 关键字简化实现，自动管理迭代状态
2. 代码复杂度
   - 迭代器-通常需要更多的代码来管理状态和迭代逻辑
   - 生成器-代码更简洁，更容易理解和维护
3. 性能与内存
   - 迭代器-性能与内存取决于实现，通常也是惰性计算
   - 生成器-使用 yield 关键字，按需生成数据，内存使用更高效
4. 使用场景
   - 迭代器-适合需要对迭代过程进行高度控制，或者需要自定义复杂的迭代逻辑时使用
   - 生成器-适合需要简洁的生成序列数据，尤其是在处理大数据或需要按需生成数据时，能够节省内存和提高性能

## 正则

- match('正则规则','字符串') 从字符串的起始位置匹配一个模式，如果从起始位置匹配成功，则返回一个匹配对象，否则返回 None。
- search('正则规则','字符串') 扫描整个字符串并返回第一个成功 的匹配。
- compile('正则规则') 编译正则表达式，返回一个正则表达式对象。后面可以跟 sub('替换字符','字符串') 替换所有匹配成功的字符串，返回替换后的字符串。
- findall() 返回所有成功的匹配，返回一个列表。
- split() 根据匹配成功的字符串将原字符串分割并返回列表。
- sub() 替换所有匹配成功的字符串，返回替换后的字符串。
